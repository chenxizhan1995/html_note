什么是Web标准
    通过web标准，可以帮助确保普适的兼容性和灵活性。
    这些标准由W3C（Word Wide Web协会, www.w3.org）。
    HTML标准基于语义学，这是通过标签定义语义的过程，如“这些文字是一个段落”，“这些对象中的HTML形成了一个表格”。
    结构与表现的分离是Web标准迈向成功的第一部。
结构与表现
    HTML = 内容
    CSS  = 表现

    “内容与表现的分离”，构建网页是必须要记住这条规则。

    其它的说法包括“意义和外观的分离”，“结构和表现的分离”。

    较真的说法是”由富含语义的结构所构成的内容与外观表现的分离“。

    HTML应当纯粹地用于构建内容，而CSS则纯粹地用于展示内容。


    这意味着，font标签已经初具，table标签面向表格数据而不是布局，有标题时应当标记为标题而不是使其看起来像，这还意味着，强调文字时要用到em标签而不仅仅是CSS修饰。

    一些结构与表现分离的范例

    范例                      结构/表现               HTML/CSS
    --------------------------------------------------------------------------------
    标题                      结构                     HTML
    标题的字号                表现                     CSS
    段落                      结构                     HTML
    段落中文字的颜色          表现                     CSS
    插图表格                  结构                     HTML
    表格单元格的边框          表现                     CSS
    图片，比如证件照          结构                     HTML
    图片，比如装饰背景        表现                     CSS
    一组导航链接              结构                     HTML
    页面上导航链接的位置      表现                     CSS
    --------------------------------------------------------------------------------
前向兼容意味着网页能够正确的显示在未来的浏览器版本中

浏览器hack
    确实存在大量的hadc，以实现为不同的浏览器使用不同的代码，但事实上，除了一个简单的框模型hack（为了解决 IE 5.x的计算错误），我们几乎不需要任何其他hack。

后向兼容
    后向兼容意味着，古老的浏览器同样能够正确的访问我们的页面。
    适当的后向兼容是好的，但过分的强调就错了。
    设计良好的HTML页面能够被古老的浏览器访问，效果可能失去，但功能不会。
